# Content Assets

This directory stores the source assets that support published case studies.

Each case study has its own dedicated directory containing supporting material such as:

- Screenshots
- Diagrams
- Workflow illustrations
- Generated reports
- Metrics
- Videos
- PDFs
- Other publication assets

These files support the authoring process and should remain organized by project.

Application assets that are served directly by the website belong in `/public`.

---

## Directory Structure

```
assets/
├── clinical-workflow-signal-audit/
├── kenya-health-dashboard/
└── remote-healthai-role-matcher/
```

Projects adapt to the publishing standard.

The publishing standard does not adapt to individual projects.
